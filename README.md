# BoardHub

실시간 1:1 보드게임을 웹에서 바로 즐길 수 있도록 구성한 보드게임 허브 프로젝트입니다.  
이 저장소는 처음의 `흑과 백` 단일 게임 구현에서 출발해, 현재는 `십이장기`까지 포함한 멀티 게임 포털 형태로 확장되었습니다.

기준일: 2026-03-08

## 사이트 주소

- 메인 접속 주소: [https://bw-genius.vercel.app](https://bw-genius.vercel.app)

`bw-genius` 도메인은 초기 서비스 이름의 흔적이고, 현재 루트 앱 타이틀과 Vercel 프로젝트명은 `BoardHub` 기준으로 정리되어 있습니다.

## 프로젝트 개요

현재 서비스는 랜딩 페이지에서 로그인과 닉네임 설정을 마친 뒤, 원하는 게임을 선택해 바로 입장하는 구조입니다.  
공통 인증과 공통 허브 UI 위에 각 게임별 룸, 상태 머신, 전적, 실시간 동기화 로직이 분리되어 올라가 있습니다.

현재 공개 상태:

- 플레이 가능: `흑과 백`
- 플레이 가능: `십이장기`
- 개발 예정: `러브 레터`

## 현재 개발 현황

### 공통 플랫폼

- Google OAuth 기반 로그인
- 공용 닉네임 설정 및 중복 방지
- 게임 선택형 랜딩 페이지
- 공통 `Room Lobby` 패턴
  - 방 만들기
  - 6자리 코드 입장
  - 게스트 준비
  - 호스트 시작
  - 대기 / 플레이 / 종료 3상태 전환
- Supabase Realtime 기반 실시간 반영
- Realtime 누락 상황을 보완하는 보조 폴링 동기화
- Vercel 배포 기반 운영
- Google AdSense 스크립트 및 배너 영역 반영

### 흑과 백

`흑과 백`은 1:1 실시간 심리전 게임으로 구현되어 있습니다.

- 최대 9라운드 진행, 5승 선취 시 즉시 종료
- 1과 9의 역전 규칙 포함
- 라운드별 선/후공 진행
- 타일 제출, 사용 타일 소진 처리
- 결과 공개 시 승/패/무만 노출
- 실시간 점수판, 라운드 상태, 공개 로그 반영
- 게임 종료 후 라운드별 숫자 공개 보드 제공
- 플레이 중 이탈 시 기권 처리 및 전적 반영
- 종료 후 `Room으로 복귀` / `게임 나가기` 동선 정리
- BGM 및 SFX 적용

백엔드 기준으로는 `bw_profiles`, `bw_rooms`, `bw_rounds_public`, `bw_submissions` 구조와 전용 RPC 흐름을 사용합니다.

### 십이장기

`십이장기`는 같은 허브 안에 추가된 두 번째 플레이 가능 게임입니다.

- 3열 x 4행 보드 기준 구현
- 방 생성, 코드 입장, 준비, 시작, 나가기 지원
- 말 이동과 포로 배치 지원
- 자의 후 승격 처리
- 포로 재사용 처리
- 왕 포획 승리
- 왕 침투 후 생존 승리(`try`) 처리
- 수순 로그와 최종 보드 상태 확인
- 기권 종료 처리와 종료 사유 표기
- 게임별 전용 전적 집계

백엔드 기준으로는 `tj_rooms`, `tj_move_logs`, `tj_player_stats` 구조와 전용 RPC 흐름을 사용합니다.  
즉, `흑과 백`과 `십이장기`는 같은 사이트를 공유하지만 게임 상태와 데이터 구조는 분리되어 있습니다.

### 러브 레터

저장소 이름의 출발점이기도 한 `러브 레터`는 아직 플레이 가능 상태가 아닙니다.  
현재 랜딩 카드에는 `Coming Soon` 상태로만 연결되어 있으며, 실제 게임 로직은 아직 붙지 않았습니다.

## 바이브코딩 프로젝트

이 프로젝트는 정적인 사양서를 먼저 완성한 뒤 한 번에 구현한 방식보다는,  
아이디어를 빠르게 시도하고 화면과 흐름을 바로 확인하면서 다듬는 바이브코딩 방식으로 구축되었습니다.

특히 아래 요소들이 그런 방식으로 확장되었습니다.

- 단일 게임에서 멀티 게임 허브로의 전환
- 공통 로비 UX와 게임별 테마 분리
- 실시간 동기화 보완
- 게임별 룰 문서와 SQL 스키마 병행 정리
- 사운드, 이미지, 광고 영역 같은 운영 요소 추가

즉, 이 저장소는 단순한 프로토타입이 아니라, 바이브코딩 기반으로 빠르게 누적 발전해 온 실서비스형 실험 프로젝트에 가깝습니다.
